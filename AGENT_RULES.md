# Правила для Агентов по сбору данных (Research Repo)

Этот документ определяет стандарты сбора, верификации и внесения данных в репозиторий Research.

## 1. Иерархия Доверия к Источникам (Levels)

- **Level A (Железная Первичка):** Оригиналы документов, официальные заявления, судебные решения, прямые цитаты из архивов.
- **Level B (Надежный Вторичный):** Респектабельные СМИ (Time, WP и др.), официальные пресс-релизы организаций, подтвержденные биографии.
- **Level C (Косвенные Данные):** Блоги, соцсети, слухи, не подтвержденные сторонними источниками данные.
- **HOLD (Ожидание):** Данные найдены, но первичный источник не подтвержден или недоступен. Такие кейсы запрещены к использованию в публичном контенте до поднятия статуса.

## 2. Правила Внесения Данных

- **Структура:** Каждая тема должна иметь свою папку.
- **Логирование:** Все изменения статусов верификации должны заноситься в  внутри соответствующей папки.
- **Запреты:** Если кейс находится в статусе **HOLD**, агент должен четко прописать, какого именно документа/источника не хватает.

## 3. Процесс работы агента
1. Поиск данных $\rightarrow$ 2. Поиск первоисточника $\rightarrow$ 3. Присвоение уровня (A/B/C/HOLD) $\rightarrow$ 4. Фиксация в репо.
